# 【Skill 创作】靠AI芯片挣了两辆M8，我让SOLO帮我盯盘换乘：不做升降梯，做轮动

## 1、Skill简介

一个自动生成 **AI 产业链轮动日报** 的 Skill。不是告诉你「要不要卖」，而是告诉你——**高了把钱搬到哪，低了在哪接回来**。每日搜遍全网估值/资金/资讯，输出一份 8 模块 PDF：哪个赛道该下车、哪个该上车。

适合在 AI 板块挣到过钱、不想坐升降梯、想主动轮动的投资者。

## 2、使用场景

2023 年底开始定投 AI 芯片、半导体、科创板方向。一点点攒，一点点加——用了两年，账户余额凑够了第一辆 M8。

今年 AI 行情加速，半年不到又滚出了一辆。但浮盈越厚，心里越不踏实——**芯片 PE 分位飙到 88%，我坐的不是财富自由号，是升降梯。**

每天早上的状态就是——支付宝看 PE，东方财富看资金，雪球看消息，同花顺看 K 线。来回切四五个 App，脑子里全是碎片信息，最怕的是**哪天跌了没来得及跑**。

我已经从 AI 挣到了钱，不想再还回去。但我也相信 AI 远没走完——**芯片炒高了，获利撤出来，还有光模块、AI 应用、机器人、智能驾驶**……关键是：什么时候搬？往哪搬？搬多少？

我就想：能不能让 SOLO 每天帮我做这件事？我只需要打开一份 PDF，该买该卖一目了然。

## 3、创作过程

一开始只是在 SOLO 里问了一个问题：「我基金赚了钱，想在 AI 板块内部轮动，有什么建议？」

SOLO 分析了一轮，给出了上下游轮动方案。我觉得有用，就问能不能做个可视化图表。

**第一回合：一个 500 行的硬编码 HTML。** SOLO 用 ECharts 生成了桑基图、饼图、热力图，看起来挺炫。但问题马上暴露——所有数据写死在 HTML 里，第二天要用就得手动改几十处。而且手机上图形全白，ECharts 没加载出来。

我提了需求：移动端友好、纯 CSS 画图、按上中下游细分赛道。

**第二回合：改成纯 CSS 单列移动端。** 进度条、恐惧贪婪圆环、赛道热度网格全换成 CSS 实现，适配 480px 宽度。赛道从原来 4 个大类拆成了 14 个（上游 AI 芯片/HBM/光模块/服务器……中游大模型/开发平台……下游 AI 办公/机器人/智驾）。可以看了，但还差东西——我问能不能加推荐基金。

**第三回合：加推荐基金，按 ETF 和主动型区分。** 上游推 ETF（费率低适合波段），下游推主动型（选股空间大挖掘黑马）。我说 ETF 太多了，不全是我想要的，SOLO 改成了上下游区分策略。到这里 HTML 已经挺完整了，我说：把这个总结成一个 Skill。

**第四回合：变成 Skill，开始磨通用性。** 第一版 SKILL.md 写完，SOLO 立刻评测——6 处硬编码问题：基金池写了「当前建议」会过期、季度表写了「Q2（当前）」到 Q3 就乱、Step 1 有交互分支在自动化下不会触发、搜索词不够通用。我开始一轮轮改：去掉硬编码建议，改成运行时动态判断；季度表改成「当前季度」；搜索词扩宽覆盖更多 AI 子板块。还加上了 Step 1.5——每次运行先搜索最新基金上市/清盘/AI 新概念，跟知识基线对比增删。

**第五回合：HTML 也大改成数据驱动。** 既然 Skill 要做成每天跑的，HTML 里的数据也得是活的。把 500 行中的具体数值全删，改成 JS 读一个 DATA JSON 对象动态渲染。赛道从 14 个扩展到 15 个、基金从 18 只变 20 只——HTML 一行不用改。

这一步出了个 bug：header 渲染写了 `el().appendChild` 当参数传进去，结果 `TypeError: parameter 1 is not of type 'Node'`。传的是函数引用不是 DOM 节点，改了两行。

然后准备参赛。我把 Skill 丢给 SOLO 评测，它又挑出 3 个问题：Step 1.5 没说清楚哪次搜索填数值哪次做结构更新、空层基金会标错操作建议、`change` 字段没写单位。全修了。

从一句「基金赚了钱怎么轮动」开始，到最终变成一个可以挂定时任务、每天早晚自动生成 PDF 的 Skill。

## 4、使用步骤

**1. 安装 Skill**

把 `ai-sector-investor-skill` 文件夹放到 SOLO 的 skills 目录即可，Skill 会自动处理 Playwright 等依赖。

**2. 一句话触发**

在 SOLO 对话中输入：
- 「生成今天的 AI 板块投资报告」
- 「帮我分析下 AI 板块轮动」
- 「AI 板块现在哪些该减仓了」

**3. 等待输出 PDF**

SOLO 自动执行：搜索最新市场数据 → 验证赛道和基金有没有更新 → 组装 JSON → 注入 HTML 模板 → Playwright 渲染 PDF。几秒后在工作目录生成 `AI板块投资报告.pdf`。

**4. 搭配定时任务（可选）**

配合 SOLO 的定时任务功能，可设置每日早 8:30、晚 15:30 自动生成，不打开 SOLO 也能收到当天的投资报告。

## 7、总结与思考

**最满意的：** 数据驱动模板。数据天天变，但 HTML 不用改——这在 Skill 设计里太重要了，否则每天维护成本比收益还大。

**后续想加：**
- 融资融券和龙虎榜数据，资金面的颗粒度还能再细
- 「盘后复盘」模式，跟早盘做对比，看一天操作对不对
- 恐惧贪婪圆环的颜色过渡优化，目前 CSS conic-gradient 还不够顺滑

**最大的体会：**

两年攒出第一辆 M8，半年又滚出第二辆。但我不打算指望第三辆——**我不想坐升降梯。我要换乘。**

芯片高了，把利润卸下来，搬到光模块。光模块涨透了，再搬到应用端。应用端兑现了，机器人、智能驾驶还在下面等着。

这个 Skill 不是什么高大上的东西，它就帮我做一件事：**别在同一个地方坐到底，该下车就下车，该换车就换车。**

从 AI 挣到钱，用 AI 管钱，在 AI 赛道间做轮动——献身于 AI，获利于 AI。我希望这个 Skill 能帮更多在 AI 板块挣到钱的人，**不做升降梯，做换乘。**
